# Sales Analytics Dashboard

An end-to-end sales analytics pipeline and executive dashboard built in Python.

This project processes raw transactional sales data, performs structured quarterly analysis, identifies key performance drivers, and presents findings through an interactive Streamlit dashboard.

The objective is to simulate a real-world business analytics workflow from data ingestion to executive decision support.

---

## Overview

This project demonstrates:

- Data ingestion and cleaning  
- Feature engineering for time-based analysis  
- Quarter-over-quarter performance comparison  
- Driver contribution analysis (category / region / segment)  
- Automated executive markdown report generation  
- Interactive business dashboard visualization  

It reflects how analytics teams structure analytical pipelines in professional environments.

---

## Business Context

Organizations frequently need to answer:

- Why did performance change quarter-over-quarter?  
- Which product categories or regions drove growth?  
- Are trends seasonal or structural?  
- Where should strategic focus be placed?  

This project provides structured analytical answers to those questions.

Separation of concerns improves:

- Maintainability

- Clarity of logic  
- Scalability of analysis  

The dashboard layer never performs business logic вЂ” it consumes structured outputs generated by the analytics pipeline.

## Example Insight (Superstore Dataset)

Using the Superstore dataset:

- Q4 2017 sales increased **42.7%** compared to Q3.  
- Growth was primarily driven by:
  - Technology (+54.6%)  
  - Furniture (+60.5%)  

This indicates strong seasonal Q4 performance, likely influenced by holiday demand patterns.
